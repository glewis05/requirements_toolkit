Manual Referral Entry & Patient Invitation

User Story

As a user (provider, genetic assistant, or referral coordinator), I want to manually create a referred patient record in the provider portal and send an invitation so the patient can complete risk assessments to determine referral eligibility.

Acceptance Criteria

  1. User can search EHR via live FHIR for patient; if match found, demographics auto-populate
  2. If no FHIR data, required fallback fields are: Name, DOB, Email, and Referring Provider
  3. User enters referring provider details during referral creation
  4. System sends "Referral Assessment Invitation" email using distinct template with referral-specific branding and language (e.g., "XXX referred you to the YYY program. Please complete this assessment...")
  5. Referring provider details are included in the patient invitation to build trust
  6. Referring provider is NOT automatically notified; notification occurs manually after genetics staff reviews and decides on referral
  7. System logs audit trail: user ID, referral form inputs, invitation timestamp (Part 11 compliant)

P4M-AUDIT-001

Audit Trail Requirements for Part 11 Compliance

Must Have Unscheduled Draft v1

Status Timeline

Draft
-
Internal Review
-
Client Review
-
Approved
-

User Story

As a Compliance Officer, I want all system actions to be captured in an immutable audit trail with user identification and timestamps, so that I can demonstrate regulatory compliance and investigate any access or data integrity concerns.

Acceptance Criteria

  1. Audit Entry Creation:
  2. All patient record views create audit entries with user ID, timestamp, and patient ID
  3. All patient record edits create audit entries with user ID, timestamp, before/after values
  4. All failed access attempts create audit entries with user ID, timestamp, attempted action, and denial reason
  5. All role assignments/changes create audit entries with user ID, timestamp, role changed, and target user
  6. All configuration changes create audit entries with user ID, timestamp, setting changed, before/after values
  7. All data exports create audit entries with user ID, timestamp, export type, and parameters
  8. Audit Entry Content:
  9. All audit entries include computer-generated timestamp (not user-editable)
  10. All audit entries include unique sequential identifier
  11. Timestamps use consistent timezone (UTC) with millisecond precision
  12. Audit Integrity:
  13. Audit records cannot be edited after creation
  14. Audit records cannot be deleted
  15. Audit table/log is protected from unauthorized access
  16. Audit data is backed up with same frequency as production data
  17. Audit Accessibility:
  18. Authorized users can query audit log by date range
  19. Authorized users can query audit log by user ID
  20. Authorized users can query audit log by patient ID
  21. Authorized users can query audit log by action type
  22. Audit log can be exported for compliance reporting
